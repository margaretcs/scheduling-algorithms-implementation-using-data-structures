LINKED LISTS

linked lists are ordered lists of data elements with each data element containing a link to its successor and sometimes its predecessor.

Linked lists can also be referred to a chain of nodes whereby, each node contains data and the address of the next node in the chain.
It has a head pointer which contains data and if there is no next node in the chain, it stores NULL in place of address.
This is impportant because unlike arrays which are stored in contiguous memory locations, linked lists are distributed in the memory. Therefore, an address is required inorder to know the next node in the chain.

structure of a node of a linked list

struct node
{
int data;
struct node *link;
};

SCHEDULING ALGORITHMS

Scheduling algorithm is the algorithm that the scheduler uses to make a choice of the process that should enter the Central Processing Unit.
Normally, processes compete to get into the critical regions which are scarse in the computers. Thereore, a scheduling algorithms are important inorder to make choices of which process follows which.

Examples of scheduling algorithms include:

.Shortest Job First
.Round Robin
.First Come Fist Serve
.Shortest Remaining Time Next e.t.c.

When to schedule:

.If a new process is added
.If an I/O interrupt occurs
.If a process exits
.If a process blocks

Scheduling Criteria

These are the characteristics that guide in designing a good scheduling algorithm.

Compute-bound processes spend most of their time computing while I/O bound processes spend most of their time waiting for an I/O.
