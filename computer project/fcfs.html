HOW THIS ALGORITHM WORKS

FCFS Scheduling algorithm automatically executes the queued processes and requests in the order of their arrival. It allocates the job that first arrived in the queue to the CPU, then allocates the second one, and so on.
In the above program, there are 2 buttons ,1 button group ,and a square.

  • Add Process Button
    This button is used to prompt the user to input the burst times for the various processes. On clicking it, a pop up box will appear which consists of :
  • Input button label
    This is where the user will input values.The valies are restricted to numbers only and have a minimum of 1 and maximum of 30 for the purpose of simulation.
  • Input Entry button
    On clicking this button,the inputted burst time will be moved to the empty right-most button in the button group.
  • Close Button
    This button closes the pop up form.

  • Forward Clock Button
    This button is initialy hidden but after adding the burst time of atleast one process,it shows up. It performs two tasks:
  • Movement of the burst time
    The first process to be inputted will be moved from the queue to the CPU.
  • Execution of processes in the CPU
    The CPU has 1 unit of time.Therefore,on clicking the forward clock button,it is simulated that each process will be executed by one unit of time hence reducing the burst time by one each time the button is clicked.

  • Button Group
    The button group is a simulation of a queue.There are 5 buttons in the group. The burst time values inputted in the pop up box are stored here.

  • Square
    The square is a simulation of a Central Processing Unit(CPU).This is where processes are sent for execution.It has 1 unit of time. Processes which have finished execution are removed from the CPU and the next process in the queue is moved to it after clicking the forward clock button.
