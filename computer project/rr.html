HOW THIS ALGORITHM WORKS

Round Robin Algorithm is one of the algorithms employed by process and network schedulers in computing.A round-robin scheduler generally employs time-sharing, giving each job a time slot or quantum (its allowance of CPU time), and interrupting the job if it is not completed by then. The job is resumed next time a time slot is assigned to that process. If the process terminates or changes its state to waiting during its attributed time quantum, the scheduler selects the first process in the ready queue to execute. In the absence of time-sharing, or if the quanta were large relative to the sizes of the jobs, a process that produced large jobs would be favored over other processes.
In the above program, there are 4 buttons ,1 button group ,1 storage button and a square.

  • Add Process Button
    This button is used to prompt the user to input the burst times for the various processes. On clicking it, a pop up box will appear which consists of :
  • Input button label
    This is where the user will input values.The valies are restricted to numbers only and have a minimum of 1 and maximum of 30 for the purpose of simulation.
  • Input Entry button
    On clicking this button,the inputted burst time will be moved to the empty right-most button in the button group.
  • Close Button
    This button closes the pop up form.

  • Add Time Slice Button
    This button is used to prompt the user to input the quantum for the algorithm. On clicking it, a pop up box will appear which prompts the user to input the time slice.It also consists similar buttons to that of the Add Process button as mentioned above.

  • Forward Clock Button
    This button is initialy hidden but after adding the burst time of atleast one process,it shows up. It performs two tasks:
  • Movement of the burst time
    The first process to arrive will be moved from the queue to the CPU but will only be executed according to the time slice and returned to the queue if it has not finished execution.The first process in the ready queue will then be moved to the CPU and this cycle contiunues.
  • Execution of processes in the CPU
    The CPU has 1 unit of time.Therefore,on clicking the forward clock button,it is simulated that each process will be executed by one unit of time hence reducing the burst time by one each time the button is clicked until the time slice is exhausted.

  • Button Group
    The button group is a simulation of a queue.There are 5 buttons in the group. The burst time values inputted in the pop up box are stored here.

  • Time Slice Storage Button
    The time slice values inputted in the pop up box after clicking input entry in the Add Time Slice button are stored here.

  • Square
    The square is a simulation of a Central Processing Unit(CPU).This is where processes are sent for execution.It has 1 unit of time. Processes are executed according to the time slice provided.When a process is inputted,it takes the specified time slice period and then removed from the CPU.this cycle continues for each process untill all of them have been executed.

The above shows a visual representation of how the processes are scheduled into the CPU(Central Processing Unit) following the Round Robin Sheduling Algorithm.
It is clearly seen that the first process to arrive is also the first one to be executed. However, the period it stays in the CPU is contolled by the time quantum.
If the first process is not able to complete the execution within the time quantum provided, an interrupt is generated which makes the process' execution to stop before it is moved at the back of the queue.
The scheduler selects another process from the ready queue and dispatches it to the processor for its execution. It is executed until the time Quantum does not exceed.The same steps are repeated until all the process are finished. Sometimes, processes are not stored adjacent to each other in the memory and an address is required to show the location of the next process that should get into the CPU.
Linked lists contain data and a link to the next member of the lists and thus it is a very important data structure that is used in scheduling of the processes.

IMPLEMENTATION USING A C PROGRAM

#include < stdio.h >
int main()
{
int count,j,n,time,remain,flag = 0,time_quantum;
int wait_time=0,turnaround_time = 0,at[10],bt[10],rt[10];
printf("Enter Number of Process:");
scanf("%d",&n);
remain = n;
for(count = 0;count < n;count++)
{
printf("\n Enter Arrival Time for process %d :",count+1);
scanf("%d",&at[count]);
printf("\nEnter Burst Time for Process %d :",count+1);
scanf("%d",&bt[count]);
rt[count] = bt[count];
}
printf("\n Enter Time Quantum:");
scanf("%d",&time_quantum);
printf("\n\t\t------------Grantt Chart--------------\n");
printf("\n\t\t Process|Turnaround Time|Waiting Time\n\n");
for(time = 0,count = 0;remain != 0;)
{
if(rt[count] <= time_quantum && rt[count] > 0)
{
time += rt[count];
rt[count] = 0;
flag = 1;
}
else if(rt[count] > 0)
{
rt[count] -= time_quantum;
time += time_quantum;
}
if(rt[count] == 0 && flag == 1)
{
remain--;
printf("\t\tP[%d]\t|\t%d\t|\t%d\n",count + 1, time-at[count], time-at[count]-bt[count]);
wait_time += time-at[count]-bt[count];
turnaround_time += time-at[count];
flag = 0;
}
if(count == n-1)
count=0;
else if(at[count+1] <= time)
count++;
else
count = 0;
}
printf("\n\t\tAverage Waiting Time= %f\n",wait_time*1.0/n);
printf("\n\t\tAvg Turnaround Time = %f\n",turnaround_time*1.0/n);
return 0;
}
After running the above program on a compiler, it will first ask you to enter the number of processes in the terminal.
You will then have to enter the arrival time and the burst time of each process.
Afterwards, it will output the Gantt Chart and then calculate the avarage time as well as the turn around time of the processes.
